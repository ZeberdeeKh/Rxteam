# Архітектура

Технічні рішення проєкту — у форматі ADR (MADR).

- **[adr/](adr/README.md)** — журнал усіх рішень (дизайн-система + архітектура), з контекстом і наслідками.
- Стандарт **дизайну** (як виглядає UI) — у [docs/design-system/](../design-system/README.md).
- Поточний стан/план рефакторингу — додається після узгодження відкритих рішень (OD).

Правило: будь-яка зміна правила дизайну чи архітектури фіксується новим ADR
(див. [CONTRIBUTING](../design-system/CONTRIBUTING.md)).

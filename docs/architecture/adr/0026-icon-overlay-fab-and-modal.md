# 0026. Третій тип кнопки `ghost` + токени icon/overlay/FAB і примітив Modal

- **Статус:** Прийнято (Accepted) — рішення засновника 2026-06-16 (OD6)
- **Дата:** 2026-06-16
- **Категорія:** Дизайн-система
- **Теги:** buttons, ghost, icon, modal, overlay
- **Доповнює:** [ADR-0008](0008-exactly-two-button-kinds.md)

## Контекст і проблема
Іконкові контролі (ThemeToggle vs хрестик BugReport), контролі поверх фото (лайтбокс
GalleryGrid) та плаваюча кнопка (BugReport FAB) кожен мав власний хардкод-рядок. Модалка є
лише в BugReport і побудована поза системою. Окремо: правило «рівно 2 типи кнопок»
([ADR-0008](0008-exactly-two-button-kinds.md)) не мало слота для **нейтральних** дій —
cancel у модалці, «прикріпити файл», перемикач режиму авторизації — тож вони або хардкодились,
або помилково ставали другою праймері-кнопкою ([AuthForm.tsx:74](../../../components/auth/AuthForm.tsx#L74)).

## Розглянуті варіанти
- Лишити «рівно 2 типи» і мапити нейтральні дії на текст-лінк/file-input — відхилено: cancel/attach у модалці виглядають як службові кнопки, текст-лінк там слабкий.
- Cancel = delete (червоний) — відхилено: семантично невірно, червоний лякає на безпечній дії.
- **Додати третій тип `ghost`** — прийнято: окремий тихий стиль для нейтральних/другорядних дій.

## Рішення
Ввести **третій тип кнопки `ghost`** — `btn("ghost")`: та сама база (висота/шрифт/uppercase),
але без заливки (`bg-transparent text-gray-600 hover:bg-gray-100 hover:text-gray-800`).
Застосування: cancel у модалці, тригер «прикріпити файл» та інші нейтральні дії-кнопки.
**Чиста навігація** (перемикач режиму «увійти↔реєстрація») лишається текстовим `ui.link`, не кнопкою.
Правило кнопок тепер: **3 типи** — `action` / `delete` / `ghost` (доповнення до [ADR-0008](0008-exactly-two-button-kinds.md)).

Додатково: токени `ui.iconBtn`, `ui.overlayBtn`/`ui.overlayIconBtn`, `ui.fab` для іконкових,
on-media та плаваючих контролів; винести компонент `Modal` (бекдроп + контейнер `rounded-lg`
+ шапка/тіло/футер). `ui.fab` лишається єдиним санкціонованим `dark:` ([ADR-0007](0007-dark-variant-exception-bugreport-fab.md)).

## Наслідки
- ✅ Нейтральні дії мають власний передбачуваний вигляд; кінець хардкоду кнопок/модалок.
- ✅ Іконкові/overlay/floating контролі — з єдиного джерела.
- ⚠️ Зміна ідентичності «рівно 2 кнопки» → «3 кнопки». `ghost` навмисно тихий, щоб не конкурувати з `action`; ревʼю має слідкувати, щоб `ghost` не вживали там, де треба `action`.
- ⚠️ `btn()` тепер приймає `"action" | "delete" | "ghost"` — оновити тип і документацію.

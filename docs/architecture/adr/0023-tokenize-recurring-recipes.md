# 0023. Винести повторювані стиль-рецепти в токени

- **Статус:** Прийнято (Accepted) — рішення засновника 2026-06-16 (OD2, OD3, OD7)
- **Дата:** 2026-06-16
- **Категорія:** Дизайн-система
- **Теги:** tokens, drift, links, empty-state, forms

## Контекст і проблема
Аудит виявив, що низка рецептів вписана в JSX руками й уже «попливла»: інлайн-лінк
(`text-[var(--c-brand-text)] hover:underline`, 9+ місць, 3 варіації), порожній стан (3 стилі),
підпис поля (`ui.label` vs `ui.meta`, ~30 полів навпіл), fieldset-бокс і legend (3 способи),
чекбокс/радіо, file-input, список-картка, акцентоване число, попередження (`text-amber-600`).
Це прямо порушує [ADR-0010](0010-tokens-only-styling.md) («стилі лише через токени»).

## Розглянуті варіанти
- Лишити як є — відхилено: дрейф лише наростатиме, єдиний стандарт неможливо тримати.
- Виносити точково під час інших задач — відхилено: частина так і лишиться хардкодом.
- Один пакет токенів за цим ADR — прийнято.

## Рішення
Додати в [styles.ts](../../../components/ui/styles.ts) токени: `link`, `warnText`,
`emptyState`, `fieldLabel`, `legend`, `fieldBox`, `checkbox`, `radio`, `fileInput`,
`listCard`, `price`, `metaFaint`, `wordmark`, `successIconCircle`. Усі чинні хардкоди
мігрувати на них. Зафіксовані значення (за рішеннями):
- **OD2:** `fieldLabel` = `mb-1 block text-sm font-medium text-gray-700` (text-sm для основних форм; `ui.meta` лишається лише для підпідписів у вкладених fieldset).
- **OD3:** `metaFaint` = `text-xs text-gray-400` — окремий *третинний* рівень тексту (ключі налаштувань, `@username`).
- **OD7:** `wordmark` зберігає вагу `font-extrabold` (800) як виняток лише для лого.

## Наслідки
- ✅ Той самий елемент виглядає однаково всюди; дрейф фізично неможливий.
- ✅ Зміна вигляду — в одному місці.
- ⚠️ Разова міграція ~20 файлів (рефакторинг).

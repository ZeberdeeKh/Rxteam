# 0009. Лише приглушені семантичні кольори для статусів

- **Статус:** Прийнято (Accepted)
- **Дата:** 2026-06-16
- **Категорія:** Дизайн-система
- **Теги:** semantic-colors, badges, dark-theme, design-tokens

## Контекст і проблема
Яскраві, насичені кольори статусів (чисті `green-500` / `red-500`) дисонують із приглушеною хакі-палітрою бренду і виглядають різко, особливо у темній темі. Семантика статусів, бейджів і банерів (success / danger / warning) має лишатися м'якою та узгодженою з оливково-мілітарним стилем. Водночас потрібен один насичений акцент для деструктивної дії, але він не повинен «протікати» у бейджі.

## Розглянуті варіанти
- Використовувати стандартні насичені тейлвінд-кольори (`green-500` / `red-500`) для всіх статусів — відхилено: «кислотні» кольори б'ються з брендом і темною темою.
- Хардкодити плашки/банери поточними класами на кожній сторінці — відхилено: веде до візуального дрейфу та неможливості єдиного стандарту.
- Один насичений колір для всього (і кнопок, і бейджів) — відхилено: насиченість доречна лише для кнопки видалення, не для індикаторів стану.

## Рішення
Семантика статусів / бейджів / банерів (success / danger / warning) використовує ВИКЛЮЧНО приглушені токени — м'які пастелі у світлій темі та напівпрозорі тінти у темній, — кожен із парою `-bg` / `-fg` (і `-soft` для тексту дельти). Жодних яскравих / «кислотних» кольорів. Єдиний насичений виняток — `--c-danger-solid` (`#dc2626`), зарезервований винятково для КНОПКИ видалення (це кнопка, а не бейдж). Усі бейджі мають єдину форму-пігулку через `badgeClass()`.

## Наслідки
- ✅ Узгоджений, спокійний вигляд статусів у світлій і темній темі, без різких контрастів із хакі-брендом.
- ✅ Єдине джерело правди для плашок: `badgeClass()` гарантує однакову форму-пігулку і кольори без хардкоду по сторінках.
- ✅ Насичений червоний лишається лише на деструктивній дії, тож «небезпека» візуально не розмивається на індикатори стану.
- ⚠️ Нові статуси/семантику можна додавати лише через відповідні токени (`-bg` / `-fg` / `-soft`) у `globals.css`, а не довільними тейлвінд-класами.
- ⚠️ Будь-яка потреба в яскравішому акценті потребує окремого рішення — `--c-danger-solid` не можна перевикористовувати у бейджах.

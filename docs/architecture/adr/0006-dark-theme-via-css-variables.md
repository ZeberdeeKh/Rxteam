# 0006. Темна тема через CSS-змінні, без dark:-варіантів

- **Статус:** Прийнято (Accepted)
- **Дата:** 2026-06-16
- **Категорія:** Дизайн-система
- **Теги:** dark-theme, css-variables, theming, tailwind

## Контекст і проблема
Базову палітру було перенесено з проєкту «Kalkulator» у вигляді CSS-змінних. Підтримка темної теми через Tailwind-варіант `dark:` на кожному елементі подвоїла б кількість класів і була б схильною до помилок. Потрібен механізм, який перемикає кольори централізовано, без ручного дублювання класів по кожному компоненту.

## Розглянуті варіанти
- **`dark:`-варіанти на кожному елементі** — відхилено: подвоєння класів, високий ризик пропустити елемент і неузгодженості.
- **Окремі CSS-файли/теми, що повністю підмінюють стилі** — відхилено: дублювання правил, складніше підтримувати єдину джерело-істину.
- **Перемикання теми через медіа-запит `prefers-color-scheme`** — відхилено: не дає користувачу явного вибору й не зберігає його; замість цього обрано клас `.dark` із localStorage.

## Рішення
Темну тему реалізовано прив'язкою Tailwind-кольорів `gray-*` і `white` до CSS-змінних (`--c-gray-*`, `--c-white`), які перевизначаються під `html.dark`; семантичні токени (`success`/`danger`/`warning`) перемикаються так само. Авторам ЗАБОРОНЕНО додавати `dark:`-варіанти для кольору — вони використовують gray-токени, які перемикаються автоматично. `darkMode` = `'class'`; клас `.dark` встановлюється до рендеру вбудованим inline-скриптом, щоб уникнути спалаху світлої теми. Нейтральну палітру навмисно зафіксовано (ніколи не перемикається) для тексту на кольорових кнопках.

## Наслідки
- ✅ Кількість класів не подвоюється: кольори перемикаються централізовано через CSS-змінні, а не через `dark:` на кожному елементі.
- ✅ Немає спалаху світлої теми при завантаженні — клас `.dark` виставляється inline-скриптом ще до першого рендеру.
- ✅ Узгодженість: усі сірі та семантичні відтінки гарантовано перемикаються разом, бо живуть в одному джерелі (`--c-gray-*`, семантичні токени).
- ⚠️ Заборона на `dark:` для кольору обмежує авторів — будь-яке перемикання кольору має йти лише через токени; винятки (наприклад, фіксована нейтральна палітра для тексту на кнопках) треба свідомо документувати.
- ⚠️ Inline-скрипт ініціалізації теми додає зв'язок між розміткою (`layout.tsx`) і ключем у localStorage (`rxteam-theme`); зміна ключа потребує синхронного оновлення в обох місцях.

# 0004. Єдиний токен --c-brand-text для всього хакі-тексту

- **Статус:** Прийнято (Accepted)
- **Дата:** 2026-06-16
- **Категорія:** Дизайн-система
- **Теги:** design-tokens, theming, brand-color, css-variables

## Контекст і проблема
Раніше по сайту ризикували розповзтися кілька відтінків хакі (`text-brand`, `text-brand-dark`, `text-brand-light`), розкиданих вручну по сторінках. Це призводило до неузгодженості між логотипом, заголовками, посиланнями та бейджами. Окремо це ламалося при перемиканні на темну тему, бо фіксований відтінок не адаптувався під фон. Потрібне було єдине джерело правди для «зеленого» (хакі/оливкового) тексту.

## Розглянуті варіанти
- **Кілька утиліт `text-brand` / `text-brand-dark` / `text-brand-light`** — відхилено: множинні відтінки розповзаються, важко тримати узгодженість і немає адаптації під тему.
- **Хардкод hex-кольору хакі на місцях** — відхилено: дублювання, неможливо змінити централізовано, ламається в темній темі.
- **Окремі токени для логотипу, заголовків, посилань і кнопок** — відхилено: зайва фрагментація, ризик розсинхрону між елементами одного бренду.

## Рішення
Рівно одна CSS-змінна `--c-brand-text` є джерелом усього хакі/оливкового тексту на сайті — логотип/вордмарк, кожен заголовок, посилання, текст пунктів меню (активний + hover), брендові бейджі та заливка action-кнопки (через `--c-action-*`, що дорівнює `--c-brand-text`). Токен є тема-залежним (`#545331` у світлій темі / `#9d9b66` у темній). Автори зобов'язані використовувати `text-[var(--c-brand-text)]` і ніколи не застосовувати `text-brand` / `text-brand-dark` / `text-brand-light`.

## Наслідки
- ✅ Єдине джерело правди для брендового тексту: один рядок змінює колір логотипа, заголовків, посилань, меню, бейджів і кнопок одночасно.
- ✅ Узгодженість бренду гарантована між усіма елементами, де є хакі-текст.
- ✅ Автоматична адаптація під світлу/темну тему без `dark:`-варіантів на кожному елементі.
- ⚠️ Жорстке правило для авторів: будь-який хакі-текст лише через `text-[var(--c-brand-text)]`; старі утиліти `text-brand*` заборонені й потребують дотримання/перевірки в рев'ю.
- ⚠️ `--c-action-*` навмисно прив'язаний до `--c-brand-text`, тож зміна брендового відтінку автоматично змінює і колір action-кнопок — це треба враховувати при майбутніх правках палітри.

# 0012. Згортувані блоки на нативному <details>, без JS

- **Статус:** Прийнято (Accepted)
- **Дата:** 2026-06-16
- **Категорія:** Дизайн-система
- **Теги:** collapsible, server-components, native-html, forms

## Контекст і проблема
Згортувані рядки (гравці, ролі, групи налаштувань) мають працювати в Server Components без клієнтського JS. Водночас поля форм усередині згорнутих секцій повинні нормально надсилатися на submit, адже сторінка налаштувань — це одна спільна форма. Потрібен примітив, який не покладається на гідрацію і не «викидає» приховані поля з відправки.

## Розглянуті варіанти
- Клієнтський React-компонент зі станом `useState` (`'use client'`) — додає JS, не працює у чистих серверкомпонентах.
- Умовний рендер вмісту за станом (монтувати/демонтувати при розкритті) — приховані поля форми не потрапляють у submit.
- Сторонній accordion із UI-бібліотеки — суперечить курсу на власноруч написаний UI без зовнішніх компонентних бібліотек.

## Рішення
Будувати примітив `Collapsible` на нативному HTML-елементі `<details>`/`<summary>` (клас `rx-collapse`), без клієнтського JS. Згорнутий вміст лишається в DOM (`display:none`), тож вкладені поля форми надсилаються при submit. Нативний маркер ховаємо, а шеврон обертаємо засобами CSS на `[open]`. Подібні згортувані списки мають перевикористовувати цей примітив.

## Наслідки
- ✅ Працює в Server Components без жодного клієнтського JS і без гідрації.
- ✅ Поля форми всередині згорнутих секцій коректно надсилаються на submit (спільна форма налаштувань).
- ✅ Єдиний, перевикористовуваний примітив із симетричними відступами замість розрізнених реалізацій.
- ⚠️ Поведінка прив'язана до семантики `<details>` (миттєве показ/приховання) — складніших анімацій розкриття без додаткового CSS/JS не отримати.
- ⚠️ Вміст завжди лишається в DOM навіть згорнутим, тож для дуже великих списків це не дає економії на рендері.

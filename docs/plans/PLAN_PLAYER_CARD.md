# План: Публічна картка гравця (Player Card)

Статус: **дизайн зафіксовано, до реалізації.** Фіча за прапорцем, default OFF.

## Суть
У кожного гравця з позивним є **публічна сторінка-картка** `rxteam.pl/u/<позивний>` з його статою
(позивний, ранг, ігри, надійність, місце в топі, ачивки). Нею діляться — і це приводить нових людей
у команду. Все крутиться навколо **сайту** (він самодостатній), бот — лише додатковий канал.

## Зафіксовані рішення
1. **Публічна сторінка `/u/<позивний>` — ядро фічі.** Показує картку гравця, а навколо неї —
   посилання назад на сайт (головна / ігри / долучитися) і **всі соцмережі команди**
   (беремо з наявного `components/site/SocialLinks.tsx` + `lib/social.ts`). Жодного CTA «йди в бота».
2. **Кабінет — без окремого перегляду картки** (вся інфа там і так є). Додаємо лише дві дії:
   - **«Поділитися профілем»** → копіює посилання на `/u/<позивний>`;
   - **«Завантажити картку»** → генерує й качає PNG-візуал (для сторіс).
3. **Рейтинг** ([components/site/RankingTable.tsx](../../components/site/RankingTable.tsx)) — клік по позивному
   відкриває `/u/<позивний>`. І десктоп-таблиця, і мобільні картки. Анонімні (без позивного) — неклікабельні.
4. **Візуал (PNG)** — SVG-шаблон + `@resvg/resvg-wasm` → PNG. Інлайнимо наявні `thumbnail_svg` бейджі
   (піксель-в-піксель наш стиль), шрифт Inter (Кирилиця+PL). Внизу картинки **QR → веде на `/u/<позивний>`**.
5. **Бот** — команда `/profile` додатково видає **посилання на `/u/<позивний>`** (і може прислати PNG).
   Авто-DM зі свіжою карткою при досягненні (нова ачивка / підвищення рангу / N-та гра) — бонус для TG,
   не обов'язковий шлях.
6. **Дані свіжі щоразу** (беруться з профілю при відкритті/генерації). **Надійність показуємо** на публічній картці.
7. Фіча за прапорцем **`feature_player_card`** (default `false`), як інші нові фічі.
8. Принцип: **сайт самодостатній** — не у всіх є Телеграм, тож усі основні дії лишаються на сайті.

## Що вже є (перевикористовуємо)
- Дані гравця — `players` (`callsign`, `rank`, `games_played`, `points_earned`, `has_patch`, `created_at`).
- Надійність — `getReliability(playerId)` ([lib/economy.ts](../../lib/economy.ts)).
- Ачивки — `player_achievements` + `achievements` (`thumbnail_svg`, `title_*`, `tier`).
- Топ/місце — `getTopPlayers()`; додамо `getPlayerPlace(id)` (count гравців з більшими `points_earned` + 1).
- Соцмережі — `lib/social.ts` + `SocialLinks.tsx` (керуються в адмінці).
- Профіль у боті — [bot.ts `/profile`](../../lib/bot.ts) (додаємо туди лінк).

## Що будуємо
- **`lib/player-card.ts`** — ядро (bot-agnostic): `getPlayerCardData(handle)`,
  `renderCardSvg(data)`, `cardToPng(svg): Buffer`.
- **`app/u/[handle]/page.tsx`** — публічна сторінка (картка + назад-на-сайт + соцмережі). Без авторизації.
- **`app/api/card/[handle]/route.ts`** — PNG-ендпоінт (кнопка «Завантажити», QR-прев'ю, бот фетчить).
- **Кабінет** — кнопки «Поділитися профілем» і «Завантажити картку».
- **RankingTable** — позивний → `<Link href="/u/<...>">`.
- **bot.ts** — лінк у `/profile` + авто-DM хуки в `notify-achievement.ts` / `buyRank` / чек-ін.
- **admin-settings** — тумблер `feature_player_card` (група «Функції»).
- **etap51** — seed `feature_player_card='false'`.
- **Шрифт Inter** (ttf, subset Cyrillic+Latin-ext) у репо для resvg.

## Потік (як це працює)
1. Гравець у кабінеті тисне «Поділитися профілем» → отримує лінк на `/u/<позивний>`;
   або «Завантажити картку» → PNG для сторіс.
2. Бот (для TG) присилає свіжу картку при досягненні; `/profile` дає лінк на публічну сторінку.
3. Гравець кидає картинку в сторіс (на ній QR).
4. Хтось сканує QR → потрапляє на `/u/<позивний>` → бачить картку + кнопки сайту + соцмережі →
   реєструється **на сайті** (без Телеграму).
5. У рейтингу клік по імені → та сама сторінка.
6. Дані завжди актуальні.

## Фази реалізації
- **Ф1 (ядро):** `lib/player-card.ts` (рендер SVG→PNG з QR + бейджі + Inter) + `app/u/[handle]` +
  API-ендпоінт. Прапорець OFF. Тест: відкрити `/u/<свій>` вручну, завантажити PNG.
- **Ф2 (інтеграції сайту):** кнопки в кабінеті + клік у рейтингу + лінк у боті `/profile`.
- **Ф3 (гачок):** авто-DM з карткою при ачивці / ранзі / N-й грі.

## Відкриті дрібниці (закриємо на старті)
- URL за позивним: гарантувати унікальність позивного або slugify+fallback на `id` при збігу.
- Перевірити `players.created_at` (для «з РІК») — майже напевно є.
- Кеш PNG за (гравець + хеш даних), щоб не рендерити щоразу.

# 04 — Контент, i18n та іконки

---

## 1. Тексти й регістр

- **Жодних хардкод-рядків у JSX.** Текст сайту — через `st(lang, key, vars)`
  ([lib/site-i18n.ts](../../lib/site-i18n.ts)); тексти бота — [lib/strings.ts](../../lib/strings.ts)
  ([ADR-0019](../architecture/adr/0019-trilingual-i18n-split-site-vs-bot.md)).
- **UPPERCASE — через CSS** (`uppercase` у токені), а не у вихідному рядку. У словнику
  текст у звичайному регістрі; візуальний регістр задає клас.
- Мови: **PL / UA / EN**. Мова сайту визначається на сервері (cookie `rx_lang` →
  `Accept-Language` → дефолт `uk`).

## 2. Багатомовність і розкладка

«Однакова кнопка — однакова висота, ширина за текстом» означає: верстка має терпіти
**різну довжину** PL/UA/EN. Тому:

- Кнопки не фіксуємо за шириною; текст не обрізаємо (`truncate` — лише там, де явно треба, напр. імʼя файлу в модалці).
- Багаторядковий контент (анонси, сценарії) — `whitespace-pre-line` + `leading-relaxed`.
- Не покладатися на те, що текст «вліземо в один рядок» — він перенесеться.

## 3. Формати чисел, дат, грошей

- Дати — через хелпери (`formatGameWhen` тощо), не вручну.
- Числа в таблицях/балансах — `tabular-nums` (рівні цифри).
- Акцентоване число/ціна — токен `ui.price` (`text-sm font-semibold` + brand-text), а не випадковий `font-semibold`.
- Дельти балів — `ui.posDelta` / `ui.negDelta` зі знаком `+`/`−`.

## 4. Іконки та гліфи

Зараз у проєкті **два** підходи: емодзі (📅 📍 💰 ⭐ 🎖️ 🎮) у контенті та inline-SVG
(хрестик, спінер, скріпка, шеврон) у `BugReport`/`Collapsible`/`GalleryGrid`.

Правила:
- **Інтерфейсні іконки** (закрити, розгорнути, спінер) — inline-SVG `currentColor`, розмір 14–18px, через токен `ui.iconBtn` для клікабельних.
- **Розмір гліфа — не текст.** Емодзі/SVG-гліф розміром `text-2xl`/`text-4xl` — це розмір *іконки*, він поза типографічною шкалою (виняток, [05](05-exceptions.md)).
- Шеврон згортання — `rx-chevron` (обертається на `[open]`).

### Політика емодзі-індикаторів

> ✅ **OD4 (вирішено).** Заводимо **єдиний набір констант** (напр.
> `GLYPH = { date:"📅", place:"📍", points:"⭐", balance:"💰", rank:"🎖️", game:"🎮", … }`
> у `components/ui`) і вживаємо **однаково в усіх подібних місцях** (home, GameCard, кабінет,
> рейтинг). Декоративні емодзі в текстах анонсів бота — поза цим правилом. Розмір гліфа —
> поза типографічною шкалою. [ADR-0027](../architecture/adr/0027-emoji-glyph-policy.md)

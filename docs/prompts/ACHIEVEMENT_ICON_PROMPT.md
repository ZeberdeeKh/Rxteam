# Промт — генерація SVG-іконок ачивок (медалі)

> **Призначення.** Єдиний стиль іконок для ачивок RX Team: кожна іконка — це **медаль**, чий
> метал відповідає рівню складності, обрамлена **лавровим вінком**, а в центрі — **символ теми**
> конкретної ачивки. Цей файл — канонічна інструкція: коли просять «зроби іконку для ачивки …»,
> читай і слідуй цьому файлу (правило закріплено в [`CLAUDE.md`](../../CLAUDE.md)).

## Як викликати

- Тегнути цей файл і описати ачивку, або просто: «зроби іконку для ачивки `<code>`, рівень `<tier>`, тема: …».
- Я (асистент) генерую готовий SVG-файл, зберігаю його, а ти вантажиш через
  `/admin/achievements` → рядок ачивки → **AchievementIconUploader** (зберігається як base64 data-URL
  у `achievements.thumbnail_svg`).

## Що мені треба знати про ачивку

1. **`code`** — для імені файлу (напр. `deploy_50`).
2. **`tier`** — `easy | mid | hard | legendary` → визначає **метал** (див. таблицю).
3. **`title` / тема** — щоб намалювати центральний символ (про що ачивка).

## Контракт виводу (жорсткі правила)

- Один **самодостатній** `<svg>`, `viewBox="0 0 200 200"`, `width="200" height="200"`.
- `role="img"` + змістовний `aria-label` (укр., з назвою ачивки і металом).
- **Заборонено:** `<script>`, обробники `on*`, зовнішні `href`/`url(...)`/`<image>` на віддалені ресурси,
  `<foreignObject>`. Усе — інлайн (вантажиться як data-URL у `<img>` → XSS-safe, але без скриптів усе одно).
- Локальні `<use href="#id">` на власні `<defs>` — дозволено (фрагмент того ж документа).
- Розмір ≤ 50 KB (ліміт аплоадера). Реально медаль ≈ 2–6 KB — не вставляй растрові дані.
- Без зовнішніх шрифтів. Якщо потрібен текст — лише як `<path>` або системний `font-family` sans-serif
  (краще уникати тексту: іконка читається в 28px).

## Метал за рівнем (`tier` → палітра)

Стопи градієнта диска йдуть від світлого відблиску (0%) до тіні (100%).

| tier | метал | диск (стопи 0% → 40% → 76% → 100%) | обідок (rim) | центр-поле (0%→100%) | символ-заливка | лист лавра (0%→100%) |
|------|-------|------------------------------------|--------------|----------------------|----------------|----------------------|
| `easy` | **бронза** | `#f0c9a0` `#cd8a4e` `#a8652f` `#6e3f1c` | `#4f2c12` | `#b06a34` → `#7a4a22` | `#ffe7cf` | `#e0b07a` → `#8a5a28` |
| `mid` | **срібло** | `#ffffff` `#e8edf2` `#c2cbd4` `#8b97a3` | `#5b6670` | `#cdd5dd` → `#9aa6b2` | `#ffffff` | `#dfe6ec` → `#9aa6b2` |
| `hard` | **золото** | `#fff7d6` `#f4d35e` `#d4af37` `#9a7d1f` | `#6e5a14` | `#e9c64e` → `#b8932b` | `#fff4cf` | `#e7d27a` → `#b08d2a` |
| `legendary` | **червоний/рубін** (золотий обідок) | `#ffd1d1` `#ef5350` `#c62828` `#7f1212` | `#d4af37` (золото) | `#d23b3b` → `#8f1717` | `#ffe1b3` | `#e7d27a` → `#b08d2a` |

Примітка: для `legendary` обідок і лавр — **золоті** (метал «вище за золото»); диск — рубіновий.

## Анатомія медалі (однакова для всіх)

Шари знизу вгору (координати у `viewBox 0 0 200 200`, центр диска `100,100`):

1. **Лавровий вінок** — ліва гілка `#wreath-l`, права = її дзеркало `matrix(-1 0 0 1 200 0)`.
   Стебло — крива `<path>`, листя — `<use href="#lf">` уздовж нього (метал-листя).
2. **Вузол** унизу (`circle` r≈5.5), де гілки сходяться.
3. **Диск** — `circle r=60 fill="url(#metal)"`, обідок `stroke rim width≈3.5`, тонкий світлий
   внутрішній блік `r=52 stroke=#fff opacity .25`.
4. **Заглиблене поле** — `circle r=46 fill="url(#field)"` (трохи темніше за диск, для контрасту символу).
5. **Символ теми** — у **safe-зоні: коло r≈40 у центрі (100,100)**. 1–2 тони, товстий контур (`rim`),
   `stroke-linejoin="round"`. Має читатись у 28px (без дрібних деталей < 2px).

## Символ (центр) — правила

- Простий силует, що символізує суть ачивки (напр.: `deploy_*` → чобіт/слід; `recruiter` →
  два силуети/рукостискання; `dawn_patrol` → сонце над горизонтом; `iron_discipline` → щит/кулак;
  `first_contact` → ціль/радар; перемога → зірка/кубок).
- Заливка = «символ-заливка» з таблиці; контур = `rim` металу.
- Тримай у safe-зоні r≈40; не торкайся внутрішнього обідка поля.

## Процес (чек-ліст для мене)

1. Визначити `tier` → взяти палітру металу.
2. Підставити стопи у `#metal`, `#field`, `#leaf`, кольори `rim`/символу.
3. Намалювати центральний символ за темою ачивки в safe-зоні.
4. Скласти `aria-label`: «Досягнення: `<title>` (рівень: `<метал укр.>`)».
5. Зберегти файл у **`assets/achievement-icons/<code>.svg`** (джерело в репо).
6. Нагадати: завантажити через `/admin/achievements` (аплоадер).

## Еталон (золото, `tier=hard`) — повний приклад

Це базовий шаблон. Для іншого рівня — заміни лише стопи градієнтів і кольори `rim`/символу за
таблицею; геометрію диска/вінка/safe-зони лиши незмінною. Символ (зірку) заміни на тему ачивки.

```svg
<svg xmlns="http://www.w3.org/2000/svg" viewBox="0 0 200 200" width="200" height="200"
     role="img" aria-label="Досягнення: приклад (рівень: золото)">
  <defs>
    <!-- Метал диска (hard → золото). Заміни стопи на потрібний метал з таблиці. -->
    <radialGradient id="metal" cx="42%" cy="34%" r="72%">
      <stop offset="0%"   stop-color="#fff7d6"/>
      <stop offset="40%"  stop-color="#f4d35e"/>
      <stop offset="76%"  stop-color="#d4af37"/>
      <stop offset="100%" stop-color="#9a7d1f"/>
    </radialGradient>
    <!-- Заглиблене центральне поле. -->
    <radialGradient id="field" cx="50%" cy="42%" r="60%">
      <stop offset="0%"   stop-color="#e9c64e"/>
      <stop offset="100%" stop-color="#b8932b"/>
    </radialGradient>
    <!-- Лист лавра (метал-зелень для золота). -->
    <linearGradient id="leaf" x1="0" y1="0" x2="0" y2="1">
      <stop offset="0%"   stop-color="#e7d27a"/>
      <stop offset="100%" stop-color="#b08d2a"/>
    </linearGradient>
    <!-- Один лист: база в (0,0), кінчик угору; переюзаємо через <use>. -->
    <path id="lf" d="M0,0 C 6,-3 8,-11 0,-17 C -8,-11 -6,-3 0,0 Z"/>
  </defs>

  <!-- ЛАВРОВИЙ ВІНОК: ліва гілка; права — дзеркало. -->
  <g id="wreath-l" fill="url(#leaf)" stroke="#7a611d" stroke-width="0.6">
    <path d="M58,166 Q 33,120 50,68" fill="none" stroke="#7a611d" stroke-width="2.2"/>
    <use href="#lf" transform="translate(54,150) rotate(-40) scale(0.95)"/>
    <use href="#lf" transform="translate(46,134) rotate(-28)"/>

  <use href="#wreath-l" transform="matrix(-1 0 0 1 200 0)"/>

  <!-- Вузол, де гілки сходяться. -->
  <circle cx="100" cy="172" r="5.5" fill="url(#metal)" stroke="#7a611d" stroke-width="1"/>

  <!-- ДИСК МЕДАЛІ. -->
  <circle cx="100" cy="100" r="60" fill="url(#metal)"/>
  <circle cx="100" cy="100" r="60" fill="none" stroke="#6e5a14" stroke-width="3.5"/>
  <circle cx="100" cy="100" r="52" fill="none" stroke="#ffffff" stroke-opacity="0.25" stroke-width="1.6"/>
```

## Параметричний шаблон (плейсхолдери в `{ }`)

```
#metal стопи: {disk_0} {disk_40} {disk_76} {disk_100}
#field стопи: {field_0} → {field_100}
#leaf  стопи: {leaf_0} → {leaf_100}
обідок (всі stroke rim): {rim}
символ fill: {symbol_fill}, символ stroke: {rim}
aria-label: "Досягнення: {title} (рівень: {metal_uk})"
центральний <path>/<g>: {symbol_geometry у safe-зоні r≈40 @ 100,100}
```

Усе інше (диск r=60, поле r=46, вінок, вузол) лишається без змін — це і є «єдиний стиль».
